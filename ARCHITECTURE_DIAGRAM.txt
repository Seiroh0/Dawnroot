RUNTIME NODE HIERARCHY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Root (SceneTree)
│
├─ Main (Node2D) ← From main.tscn
│  └─ [Minimal, just entry point]
│     └─ main.gd (prints initialization message)
│
└─ GameRoot (Node) ← Autoload Singleton
   │  └─ game_root.gd (manages lifecycle)
   │
   └─ GameWorld (Node2D) ← Created programmatically
      │  └─ game_world.gd (core game logic)
      │
      ├─ BackgroundLayer (CanvasLayer, z:-20)
      │  └─ Background (ColorRect)
      │     └─ Dark blue/black background color
      │
      ├─ LevelRoot (Node2D)
      │  │
      │  ├─ Projectiles (Node2D, z:10)
      │  │  └─ [GunbootBullet instances]
      │  │     └─ [TurretProjectile instances]
      │  │
      │  ├─ Effects (Node2D, z:20)
      │  │  └─ [Particle effects]
      │  │     └─ [Animated sprites]
      │  │
      │  ├─ Player (CharacterBody2D)
      │  │  ├─ CollisionShape2D
      │  │  ├─ AnimatedSprite2D (visual)
      │  │  └─ Sprite2D (shadow)
      │  │
      │  ├─ Camera (Camera2D)
      │  │  └─ Follows player with smooth lerp
      │  │     └─ Implements screen shake
      │  │
      │  └─ LevelGenerator (Node2D)
      │     └─ Chunks (multiple Node2D containers)
      │        │
      │        ├─ Platforms (StaticBody2D)
      │        │  ├─ CollisionShape2D
      │        │  └─ ColorRect (visual)
      │        │
      │        ├─ Enemies (CharacterBody2D or Area2D)
      │        │  ├─ GroundEnemy (walks)
      │        │  ├─ FlyingEnemy (hovers)
      │        │  ├─ HeavyEnemy (tank)
      │        │  └─ TurretEnemy (shoots)
      │        │
      │        ├─ DestructibleBlocks (Area2D)
      │        │  └─ Break when shot
      │        │
      │        ├─ Gems (Area2D)
      │        │  └─ Collectible currency
      │        │
      │        └─ HazardSpikes (Area2D)
      │           └─ Damage on contact
      │
      ├─ GameHud (CanvasLayer)
      │  └─ UI Elements
      │     ├─ Score Label
      │     ├─ Gems Label
      │     ├─ Health Display
      │     ├─ Ammo Display
      │     ├─ Combo Display
      │     ├─ Depth Display
      │     └─ Message Label (temporary messages)
      │
      ├─ ShopUI (CanvasLayer)
      │  └─ Shop Interface
      │     ├─ Background
      │     ├─ Option Buttons
      │     └─ Description Labels
      │
      └─ AudioPlayers (AudioStreamPlayer nodes)
         ├─ stomp (procedural beep)
         ├─ damage (procedural beep)
         ├─ gem (procedural beep)
         ├─ block (procedural beep)
         ├─ shot (procedural beep)
         └─ BackgroundMusic (mp3 stream)

EXECUTION TIMELINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

T=0.000s  │ Godot engine starts
          │
T=0.001s  │ Read project.godot
          │ ├─ Parse configuration
          │ └─ Queue autoloads
          │
T=0.002s  │ Initialize autoload: GameRoot
          │ ├─ Instantiate game_root.gd as Node
          │ ├─ Add to SceneTree as singleton
          │ └─ Call GameRoot._ready()
          │    ├─ Load save data
          │    └─ Call _start_run()
          │       ├─ Create GameWorld instance
          │       ├─ Add as child to GameRoot
          │       └─ Connect signals
          │
T=0.003s  │ Load main scene: main.tscn
          │ ├─ Instantiate Main (Node2D)
          │ ├─ Attach main.gd script
          │ ├─ Add to SceneTree
          │ └─ Call Main._ready()
          │    └─ Print initialization message
          │
T=0.004s  │ GameWorld._ready() executes
          │ └─ Call _build_world()
          │    ├─ Create background
          │    ├─ Create player at (0, 120)
          │    ├─ Create camera
          │    ├─ Create level generator
          │    ├─ Create HUD
          │    ├─ Create shop UI
          │    ├─ Setup audio
          │    └─ Start background music
          │
T=0.050s  │ First frame rendered
          │ ├─ Background visible
          │ ├─ Player visible
          │ ├─ Initial chunks generated
          │ └─ HUD displayed
          │
T=0.016s+ │ Game loop running (60 FPS)
          │ └─ _physics_process() every frame:
          │    ├─ Update player physics
          │    ├─ Update enemies
          │    ├─ Generate new chunks
          │    ├─ Update camera position
          │    ├─ Process collisions
          │    └─ Update screen shake


ARCHITECTURE BENEFITS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✓ Autoload Singleton Pattern
  └─ GameRoot persists across scene changes
     └─ Future: Can load menu scenes without losing progress

✓ Programmatic Node Creation
  └─ Perfect for procedural generation
     └─ Flexible configuration at runtime

✓ Clean Separation of Concerns
  ├─ main.tscn: Entry point only
  ├─ GameRoot: Lifecycle & meta-progression
  └─ GameWorld: Actual gameplay logic

✓ Minimal Scene Overhead
  └─ No complex .tscn dependencies
     └─ Easy to modify and extend

✓ Signal-Based Communication
  └─ Loose coupling between systems
     └─ Easy to add new features
